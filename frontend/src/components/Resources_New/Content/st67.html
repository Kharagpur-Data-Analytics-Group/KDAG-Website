Weight Initialisation

The optimization landscape of a deep neural network is non-convex, high-dimensional, and filled with local minima, saddle points, and flat regions. The starting point—determined by the initial weights—strongly influences the path the optimizer takes through this complex loss surface.

If the weights are initialized poorly, gradients may either vanish or explode, leading to stalled training or divergence. On the other hand, good initialization brings the model closer to an optimal solution.

A Simple Example: Equal Weight Initialization

Suppose all weights in a network layer are initialized to the same value. This creates a symmetry problem—all neurons in the layer produce the same output and receive the same gradient. As a result, they evolve identically during training, defeating the purpose of having multiple neurons.

Here we can observe that forward propagation yields consistent activations for fixed weights and inputs, just as backpropagation produces identical gradients, ensuring deterministic training steps.

So, we need random initialisation for better learning.

Historical Initialization Techniques

Greedy Layerwise Unsupervised Pre Training

Before the era of ReLUs and batch norm, one common method was greedy layerwise pretraining. Here's how it worked:

For the first hidden layer, an unsupervised learning algorithm (like an autoencoder or RBM) is used to learn weights.The output from this layer is passed to the next, where another unsupervised model learns new weights.This process continues layer by layer, initializing the full network.Finally, supervised fine-tuning is done via backpropagation.

While largely outdated, this technique laid the groundwork for modern deep learning.
